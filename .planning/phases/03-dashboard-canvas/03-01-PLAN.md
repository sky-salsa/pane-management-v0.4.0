  <action>
    Create `commands/tmux.rs` with IPC commands that query and mutate tmux state via `wsl.exe -e bash -c "tmux ..."`.

    **Helper function** `run_tmux_command(script: &str) -> Result<String, String>`:
    - Run via `std::process::Command::new("wsl.exe").args(["-e", "bash", "-c", script])`
    - On Windows, use `.creation_flags(0x08000000)` (CREATE_NO_WINDOW) -- use `#[cfg(windows)]` and `std::os::windows::process::CommandExt`
    - If stderr contains "no server running" or "no sessions", return `Ok(String::new())` (not an error)
    - Otherwise check exit code and return stderr on failure

    **Query commands:**

    `list_tmux_sessions() -> Result<Vec<TmuxSession>, String>`:
    - Command: `tmux list-sessions -F '#{session_name}|#{session_windows}|#{session_attached}' 2>/dev/null`
    - Parse pipe-delimited output into TmuxSession structs

    `list_tmux_windows(session_name: String) -> Result<Vec<TmuxWindow>, String>`:
    - Command: `tmux list-windows -t {session_name} -F '#{window_index}|#{window_name}|#{window_panes}|#{window_active}' 2>/dev/null`
    - Parse into TmuxWindow structs

    `list_tmux_panes(session_name: String, window_index: u32) -> Result<Vec<TmuxPane>, String>`:
    - Command: `tmux list-panes -t {session_name}:{window_index} -F '#{pane_index}|#{pane_id}|#{pane_width}|#{pane_height}|#{pane_top}|#{pane_left}|#{pane_active}|#{pane_current_command}|#{pane_current_path}' 2>/dev/null`
    - Parse into TmuxPane structs

    `get_tmux_state(session_name: String, window_index: u32) -> Result<TmuxState, String>`:
    - Batched query using `---SESSIONS---`, `---WINDOWS---`, `---PANES---` separators (as per research Pattern 3)
    - Single wsl.exe call, parse by splitting on marker lines
    - Returns TmuxState with all three Vec fields populated

    **Mutation commands:**

    `create_pane(session_name: String, window_index: u32, direction: String) -> Result<Vec<TmuxPane>, String>`:
    - `direction` is "h" or "v" for horizontal/vertical split
    - Command: `tmux split-window -t {session}:{window} -{direction}`
    - After split, re-query panes and return updated list

    `apply_layout(session_name: String, window_index: u32, layout: String) -> Result<Vec<TmuxPane>, String>`:
    - `layout` is one of: "even-horizontal", "even-vertical", "main-horizontal", "main-vertical", "tiled"
    - Command: `tmux select-layout -t {session}:{window} {layout}`
    - After applying, re-query panes and return updated list

    `send_to_pane(session_name: String, window_index: u32, pane_index: u32, command: String) -> Result<(), String>`:
    - Command: `tmux send-keys -t {session}:{window}.{pane} '{command}' Enter`
    - Escape single quotes in command by replacing `'` with `'\''`

    `kill_pane(session_name: String, window_index: u32, pane_index: u32) -> Result<Vec<TmuxPane>, String>`:
    - Command: `tmux kill-pane -t {session}:{window}.{pane}`
    - Re-query panes after kill and return updated list

    **Register in mod.rs:**
    ```rust

    pub mod launcher;
    pub mod tmux;
    ```

    Add basic unit tests for the helper function (can't test actual tmux in CI, but test parsing logic):
    - Test `parse pipe-delimited session line` -> TmuxSession
    - Test `parse pipe-delimited pane line` -> TmuxPane
    - Test `empty output` -> empty Vec
    Extract parsing functions as separate testable functions (e.g., `parse_session_line`, `parse_pane_line`).
